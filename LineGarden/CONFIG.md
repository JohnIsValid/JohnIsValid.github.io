# 《回路花园》关卡配置说明

游戏的 50 个关卡由 `config.js` 中的 `buildLevel()` 按固定规则生成。关卡名称保存在 `LEVEL_NAMES`，元素开放节奏由 `mechanicsFor()` 控制，棋盘和符号在 `buildLevel()` 中生成。生成过程不使用随机数，因此每次打开页面都会得到相同的关卡。

## 最快的配置方法

1. 修改 `LEVEL_NAMES` 可以调整 50 个关卡的显示名称。
2. 修改 `mechanicsFor(levelNumber)` 可以调整某关启用哪些玩法元素。
3. 修改 `chapterFor()` 与 `hintFor()` 可以调整章节和提示。
4. 修改 `buildLevel()` 中的尺寸、符号数量和选点公式，可以整体调整难度。
5. 如需完全手工制作某一关，可以让 `buildLevel()` 在该关卡编号下直接返回一个关卡对象。

## 玩法开放表

| 首次出现关卡 | 元素 | 作用 |
| --- | --- | --- |
| 第 6 关 | 金色光点 | 路径必须经过全部光点。 |
| 第 12 关 | 颜色方块 | 路径需要把不同颜色分到不同区域。 |
| 第 18 关 | 计数三角 | 数量表示路径必须贴着该格子的边数。 |
| 第 24 关 | 断裂道路 | 带断口的边无法通行，必须绕路。 |
| 第 30 关 | 成对花芽 | 同色花芽在一个区域内必须恰好出现两个。 |

第 30 关会同时启用全部五种元素；第 31～50 关采用不同组合，第 45～50 关再次同时启用全部元素。

## 配置表

| 字段 | 类型 | 必填 | 示例 | 说明 |
| --- | --- | --- | --- | --- |
| `id` | 字符串 | 是 | `"first-step"` | 关卡唯一标识；发布后不要随意改，否则本地通关记录会失效。 |
| `name` | 字符串 | 是 | `"初见"` | 玩家看到的关卡名。 |
| `chapter` | 字符串 | 是 | `"萌芽"` | 顶部显示的章节名。 |
| `hint` | 字符串 | 是 | `"从左下角出发……"` | 简短提示，建议只提醒观察方向，不直接公布答案。 |
| `cols` | 正整数 | 是 | `4` | 棋盘横向格子数；节点横坐标范围为 `0～cols`。 |
| `rows` | 正整数 | 是 | `4` | 棋盘纵向格子数；节点纵坐标范围为 `0～rows`。 |
| `start` | `[x, y]` | 是 | `[0, 4]` | 起点节点。左上角为 `[0, 0]`，向右、向下递增。 |
| `end` | `[x, y]` | 是 | `[4, 0]` | 出口所在节点。 |
| `exitDirection` | 字符串 | 否 | `"right"` | 出口朝向：`right`、`left`、`up`、`down`；默认 `right`。 |
| `checkpoints` | 坐标数组 | 否 | `[[1, 4], [1, 2]]` | 路径必须经过的节点光点。 |
| `coloredCells` | 对象数组 | 否 | `{ x: 0, y: 1, color: "teal" }` | 放在格子内部；不同颜色方块必须被路径分到不同连通区域。 |
| `triangles` | 对象数组 | 否 | `{ x: 1, y: 2, count: 2 }` | `count` 为路径必须贴着该格子的边数，取值建议为 `1～3`。 |
| `blockedEdges` | 边数组 | 否 | `[[[1, 1], [2, 1]]]` | 无法经过的网格边；两个节点必须相邻。 |
| `blossoms` | 对象数组 | 否 | `{ x: 2, y: 1, color: "lavender" }` | 同色花芽在每个区域内只能恰好出现两个。 |

## 坐标规则

节点坐标用于起点、出口和光点；格子坐标用于颜色方块和三角。二者都以左上角为原点。

```text
节点： (0,0) ─ (1,0) ─ (2,0)
         │  格(0,0) │  格(1,0)
节点： (0,1) ─ (1,1) ─ (2,1)
```

一个 `4 × 4` 棋盘共有 `5 × 5` 个节点，所以起点或光点可以使用 `[4, 4]`；格子坐标最大只能是 `{ x: 3, y: 3 }`。

当 `cols` 与 `rows` 不相等时，棋盘会保持正方形格子并自动居中显示。例如 `cols: 5, rows: 2` 会生成一个横向长方形棋盘。

## 完整示例

```js
{
  id: "sample-level",
  name: "示例关卡",
  chapter: "试验田",
  hint: "经过光点，并分开两种颜色。",
  cols: 4,
  rows: 4,
  start: [0, 4],
  end: [4, 0],
  exitDirection: "right",
  checkpoints: [[1, 4], [1, 2], [3, 2]],
  coloredCells: [
    { x: 0, y: 1, color: "teal" },
    { x: 3, y: 2, color: "coral" }
  ],
  triangles: [
    { x: 1, y: 2, count: 2 }
  ],
  blockedEdges: [
    [[2, 0], [2, 1]]
  ],
  blossoms: [
    { x: 0, y: 2, color: "lavender" },
    { x: 1, y: 2, color: "lavender" }
  ]
}
```

## 可用颜色与扩展颜色

颜色方块内置 `teal`（青绿）和 `coral`（珊瑚）；花芽内置 `lavender`（淡紫）和 `sky`（天蓝）。也可以直接使用 CSS 颜色，例如 `color: "#b68cff"`。判定时颜色字符串完全相同才视为同色，建议同一颜色始终使用同一种写法。

如果要增加一个具名颜色，请同时在 `game.js` 的 `colors` 对象中加入颜色值；若只使用十六进制颜色，则无需改动脚本。

## 关卡设计建议

- 一关只引入一个新概念，下一关再将它与旧规则组合。
- 光点约束“必须去哪里”，三角约束“怎样经过”，断径约束“不能走哪里”，颜色与花芽约束“路径最终如何切分空间”。
- 先画出一条目标路径，再沿路径布置三角与光点；随后添加断径排除近路，最后用颜色方块和花芽约束区域。
- `4 × 4` 适合教学，`5 × 5` 适合组合题，移动端不建议超过 `7 × 7`。
- 提交前至少用鼠标和手机触摸各通关一次，并确认错误提示能指出违反了哪类规则。

## 判定顺序

画线过程中会即时阻止玩家经过断裂道路。抵达出口后，游戏依次检查：起点与出口 → 光点 → 三角贴边数量 → 颜色区域 → 花芽配对。发现错误后会显示对应提示，并在 1.5 秒后自动重开当前关卡。

通关进度保存在浏览器本地，键名由 `game.storageKey` 控制；修改这个值会让所有玩家在该设备上的通关记录重新开始。
