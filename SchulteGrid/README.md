# 舒尔特方格（Number Sprint）

一个无需安装依赖、打开浏览器即可使用的舒尔特方格专注力训练小游戏。玩家需要从数字 `1` 开始，按照升序依次点击棋盘中的全部数字，并尽可能缩短完成时间。

## 什么是舒尔特方格训练

舒尔特方格是一种视觉搜索与注意力练习：把一组数字随机分布在方格中，训练者保持视线尽量位于棋盘中央，同时按顺序快速找到所有数字。训练过程中需要持续观察整个棋盘，在多个视觉信息之间筛选目标，并协调注意、判断和点击动作。

它常被用于练习注意力稳定性、视觉搜索速度和阅读时的视线控制。完成时间可以作为个人训练记录，但更适合与自己在相同棋盘尺寸下的历史成绩比较，不宜简单地作为智力、视力或健康状况的评价标准。

## 经常训练的好处

在难度适当、规律练习的情况下，舒尔特方格训练可能有助于：

- **提升专注与持续注意能力**：在一段时间内始终围绕下一个目标数字完成任务。
- **提高视觉搜索效率**：练习更快地从杂乱信息中定位需要的内容。
- **增强周边视觉信息利用能力**：减少只盯着单个格子逐一查找的习惯，尝试感知更大的视野范围。
- **改善视线移动策略**：逐渐减少无目的的来回扫视，对快速阅读和信息检索有一定辅助价值。
- **锻炼反应与手眼协调**：看到目标、确认顺序并点击，需要视觉判断和动作配合。
- **获得可量化的训练反馈**：通过完成时间、错误次数和棋盘尺寸观察自己的长期变化。

训练效果因人而异。舒尔特方格适合作为日常注意力练习，但不能替代视力检查、医疗诊断或专业治疗；训练时若出现明显眼疲劳、头晕或不适，应暂停并休息。

## 功能特点

- 支持 `4 × 4`、`5 × 5`、`6 × 6`、`7 × 7` 四种棋盘尺寸
- 选择棋盘尺寸后，可继续选择多色或同色方格
- 每局随机生成数字位置；多色模式会随机安排方格颜色
- 避免连续数字在上下左右相邻，减少顺序连点带来的偶然优势
- 点击数字 `1` 时自动开始计时，计时精确到百分之一秒
- 实时显示下一个目标数字和完成进度
- 点击错误数字时提供动画与文字提示
- 完成后根据用时给出“优秀”“尚可”或“需要提高”的评级
- 同色模式下，“优秀”和“尚可”的时间上限均缩短 8 秒
- 支持重新开始当前尺寸，或返回选择其他棋盘尺寸
- 提供移动端适配、键盘焦点样式和减少动画偏好支持

## 快速开始

本项目由原生 HTML、CSS 和 JavaScript 编写，没有第三方依赖，也不需要构建。

### 方式一：直接打开

双击 `index.html`，使用现代浏览器打开即可开始训练。

### 方式二：启动本地静态服务器

如果电脑已安装 Python，可在项目目录执行：

```bash
python -m http.server 8000
```

然后访问：

```text
http://localhost:8000
```

## 游戏规则

1. 选择棋盘尺寸。
2. 选择每格不同颜色或所有格子相同颜色。
3. 找到并点击数字 `1`，此时计时开始。
4. 按照 `2、3、4……` 的顺序点击，直到完成全部数字。
5. 尽量保持视线覆盖整个棋盘，减少逐格搜索和头部移动。
6. 完成后可查看用时与评级，并选择再玩一次。

## 完成评级

评级阈值会随棋盘数字总数变化：

- **优秀**：完成秒数不超过数字总数
- **尚可**：完成秒数不超过数字总数的 `1.2` 倍
- **需要提高**：超过上述时间

例如，`5 × 5` 棋盘共有 25 个数字，25 秒内为“优秀”，30 秒内为“尚可”。

## 实现说明

- 棋盘使用 Fisher–Yates 洗牌算法随机排列数字。
- 生成布局时会检查连续数字的位置；若任意一对连续数字上下左右相邻，则重新洗牌。
- 方格颜色从五种配色中生成，并尽量避免相邻方格及连续数字使用相同颜色。
- 计时基于 `performance.now()` 和 `requestAnimationFrame()`，以获得平滑、较高精度的显示。
- 游戏状态分为尺寸选择、等待开始、进行中和已完成，避免重复计时或完成后继续操作。

## 浏览器支持

建议使用较新版本的 Chrome、Edge、Firefox 或 Safari。项目依赖 CSS Grid、CSS 自定义属性、`requestAnimationFrame()` 等现代浏览器能力。

## 训练建议

- 初次使用可从 `4 × 4` 开始，熟悉后逐步提升尺寸。
- 尽量将视线保持在棋盘中央，用周边视觉寻找目标数字。
- 记录相同尺寸下的完成时间，比单次追求速度更能反映训练效果。
- 保持准确优先，形成稳定节奏后再提高速度。
